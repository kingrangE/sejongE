# AI 티 제거 체크리스트

초안을 쓴 뒤 아래 항목으로 점검한다. 하나라도 걸리면 고친다.
목표: "성실한 개발자가 직접 정리한 README"로 읽히게.

## 문체·어조
- [ ] 이모지로 섹션 헤더를 장식하지 않았다(🚀 ✨ 🔥 등). 있어도 극히 절제.
- [ ] "혁신적인", "강력한", "원활한(seamless)", "손쉽게", "최첨단", "차세대" 같은
      마케팅·홍보 형용사를 쓰지 않았다.
- [ ] "Whether you're a beginner or an expert…", "In today's fast-paced world…"
      류의 공허한 도입부가 없다.
- [ ] 모든 섹션을 같은 길이로 대칭으로 채우지 않았다(할 말 양에 따라 비대칭이 자연스럽다).
- [ ] 불릿마다 굵은 글씨 + 콜론으로 시작하는 기계적 반복 패턴이 과하지 않다.
- [ ] 과한 em-dash(—) 남발, 지나치게 매끄러운 "삼단 병렬" 문장이 반복되지 않는다.

## 내용의 진실성
- [ ] 모든 기술 언급이 실제 의존성/코드에서 확인된다(코드에 없는 기술 언급 금지).
- [ ] 측정하지 않은 성능 수치("응답속도 50% 개선" 등)를 지어내지 않았다.
- [ ] 존재하지 않는 기능·테스트·배포 파이프라인을 있는 것처럼 쓰지 않았다.
- [ ] 실행 명령이 실제 scripts/문서와 일치한다(붙여넣으면 동작).

## 설득력(면접 관점)
- [ ] 기술 나열에 그치지 않고 "왜 이 선택을 했는가"가 최소 2~3곳에 드러난다.
- [ ] 대안을 검토한 흔적(트레이드오프)이 보인다.
- [ ] 개발을 어떤 순서로 왜 그렇게 진행했는지가 보인다.
- [ ] 한계/미완 영역을 솔직히 적었다(과장보다 신뢰를 준다).

## 형식
- [ ] 목차/배지를 형식적으로 끼워넣지 않았다(실제 가치 있을 때만).
- [ ] 기존 README의 사람이 쓴 정보(라이선스, 스크린샷, 설치 주의사항)를 보존했다.
